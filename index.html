Introduction

Welcome to this tutorial on how to develop a screen reader accessible webpage! In this tutorial, we will be covering the basics of how to create a webpage that is accessible for screen reader users. Before getting started on designing an accessible website you should download a tool like WAVE which gives you an automatic accessibility evalutation. Also be sure to familiarize yourself with the WCAG 2.1 guidelines . The purpose of this tutorial is to provide a simple explanation of how you can meet some of these guidelines.

HTML Structure

One of the most important things to consider when creating a screen reader accessible webpage is the HTML structure. Here are a few tips:

  • Use proper semantic HTML tags such as <h> and <nav> to provide structure and context for the content.
    • Tip: The use of <h> tags are very useful when trying to navigate with a screen-reader. By using <h1> for the title of the webpage and <h2> for sections under that you can
      provide a logical structure to your webpage.
    • For example:
    • <h1> Title of Page </h1>
       <h2> Section 1 </h2>
       <h2> Section 2 </h2>
        <h3> Sub-Section </h3>
         *Some content goes here*
  • Make sure that all non-text elements such as images, buttons, and links have descriptive alternative text using the alt attribute.
    • One common mistake that's often made by web developers is that they provide important information in an inaccessible way. For example providing a phone number in an image without proper alternative text.
    • Here's an example of a phone number as an image:
    • Rather than providing this important information as an image, if possible it's better to display it as text so the screen reader can convey it to the user properly.
    • Here's an example of some text styled to display the same phone number in the same way as the image but with greater screen reader accessiblity:
    • 123-456-7890
  • Similarly to images, make sure that all videos on your site are screen reader accessible. Make sure that they can be played by using just the keyboard. The user must be able to tab to the video and play it without needing to use a mouse. Another key accessibility concern with videos is closed captioning, although this doesn't have to do with screen reader accessibility, you should always provide captions for any video you have on your webpage. An easy way to create captions for your video is by uploading it to YouTube and generating automatic captions that you can then go through and edit. After uploading the video you can just link it on your webpage.
  • Another key area which is often forgetten about when it comes to screen reader accessibility is forms
    • Be sure to provide proper labels for each form item be using the <label> tag.
    • Placeholder text is not a replacement for labels as placeholders are not treated the same in screen readers. Placeholders are also not supported across all assistive techologies and are not displayed in older browsers.
    • Here's an example of form labeling:
      <label for="email">Email: </label><input type="email" id="email" placeholder="joe@example.org">
  • Provide skip links so screen reader users are able to more easily navigate the page. These can be found at the top of this webpage. Each link allows the user to navigate to a different section of the webpage without scrolling.
  • Use ARIA attributes such as role, aria-label, and aria-describedby to provide additional context and information for screen reader users.
  • One of the biggest things that screen reader users struggle with is "not knowing what they don't know." Users that utilize a screen reader won't know whether information is inaccessible to them or if its just not provided. As discussed earlier, by providing important information in images without proper labeling, screen reader users may just assume that the information is not on the page. Be sure to properly label and describe all important elements of the page so that the screen reader user is able to obtain the same information as a user not using using a screen reader.
  • Some other simple tips that will allow your website to be more accessible are:
    • Provide a light and dark mode. Some users may find content on a webpage easier to understand with a dark versus a light background so providing this simple feature can go a long way.
    • Many web developers want to make interesting dynamic websites but often forget that animations and flashing colors might hurt some users. Be sure to provide users with the option to disable animations and be provided with a static webpage easily. You can see examples of these two features implemented on this webpage.

CSS Styling

While CSS styling is not directly related to screen-reader accessibility, it can have an impact on the user experience and general accessibility. Here are a few tips:

  • Use high contrast colors to make sure that text is easy to read.
  • Make sure that font sizes are not too small and that line heights are appropriate to make the text easy to read.
  • Use CSS to hide any visual elements that are not relevant to screen reader users, but make sure that they are still accessible using keyboard navigation.

JavaScript Enhancements

JavaScript can be used to enhance the accessibility of a webpage. Here are a few tips:

  • Use JavaScript to add keyboard shortcuts for frequently used actions to make it easier for screen reader users to navigate the webpage.
  • Use JavaScript to dynamically update the content on the page without requiring a full page refresh. This can make it easier for screen-reader users to navigate and understand the content.
  • Use ARIA attributes such as aria-live and aria-atomic to make sure that any dynamically updated content is properly announced to screen-reader users.

Conclusion

By following these tips for HTML structure, CSS styling, and JavaScript enhancements, you can create a screen-reader accessible webpage that provides an excellent user experience for all users, regardless of their abilities. If you have any questions or feedback on this tutorial, please let us know!
